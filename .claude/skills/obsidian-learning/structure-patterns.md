# 文章结构模式

根据内容特点选择合适的结构模式。

## 模式1：问题-方案型

**适用场景**：教程、指南、解决方案类文章

**结构**：
```
1. 开篇：痛点共鸣（读者遇到的问题）
2. 分析：问题根源（为什么会这样）
3. 方案：解决步骤（怎么做）
4. 案例：成功证明（实际效果）
5. 结尾：行动号召（下一步）
```

**示例**：
- "为什么你的代码总是难以维护？"
- "3步解决远程团队沟通问题"

---

## 模式2：对比型

**适用场景**：评测、选择指南、观点辨析

**结构**：
```
1. 开篇：对比背景（为什么要比较）
2. 维度A：[对比项1] vs [对比项2]
3. 维度B：[对比项1] vs [对比项2]
4. 维度C：[对比项1] vs [对比项2]
5. 结论：适用建议（什么情况选什么）
```

**示例**：
- "Claude vs GPT：2024年该选哪个？"
- "React vs Vue：不同项目的最佳选择"

---

## 模式3：递进型

**适用场景**：深度分析、认知升级、教育内容

**结构**：
```
1. 表象：大家看到的现象
2. 本质：背后的真正原因
3. 规律：可复用的原理
4. 应用：如何在实践中使用
5. 升华：更高层次的思考
```

**示例**：
- "从写代码到写架构：程序员的认知升级"
- "为什么懂了很多道理却过不好这一生"

---

## 模式4：故事型

**适用场景**：个人经历、案例分析、品牌故事

**结构**：
```
1. 场景：设置情境（时间、地点、人物）
2. 冲突：遇到问题或挑战
3. 探索：尝试解决的过程
4. 转折：关键的发现或改变
5. 结局：最终结果和收获
6. 启示：普适性的道理
```

**示例**：
- "我是如何从月薪5K到年薪百万的"
- "一次失败的创业教会我的三件事"

---

## 模式5：清单型

**适用场景**：工具推荐、技巧汇总、资源整理

**结构**：
```
1. 开篇：清单的价值（为什么整理这个）
2. 分类1：[3-5个条目]
3. 分类2：[3-5个条目]
4. 分类3：[3-5个条目]
5. 结尾：使用建议或补充说明
```

**示例**：
- "2024年程序员必备的20个效率工具"
- "新手入门机器学习的10本书"

---

## 模式6：SCQA型

**适用场景**：商业报告、咨询方案、严谨论述

**结构**：
```
S - Situation（情境）：当前状态
C - Complication（复杂性）：遇到的挑战
Q - Question（问题）：核心问题是什么
A - Answer（答案）：解决方案
```

**示例**：
- "公司数字化转型方案报告"
- "市场进入策略分析"

---

## 选择指南

| 如果你的内容是... | 推荐使用 |
|------------------|---------|
| 教人怎么做 | 问题-方案型 |
| 比较两个选项 | 对比型 |
| 解释深层原理 | 递进型 |
| 讲述真实经历 | 故事型 |
| 整理多个资源 | 清单型 |
| 商业/专业分析 | SCQA型 |
