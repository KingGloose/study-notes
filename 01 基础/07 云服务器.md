# 1 域名

## 1.1 主机记录

针对这里得 “主机记录”，**一个域名往往不只对应一个服务**，所以 DNS 需要一个机制，把同一个主域名下面的不同用途区分开，“同一个域名体系下，如何给不同服务分配不同入口名”

你可以把它理解成：
- `didastone.com` 是一个总地址
- 但你的网站、接口、后台、测试环境，不可能全都混在一个名字上
- 所以要在前面加一个“标识”
于是就有了：
- `www.didastone.com`：官网
- `api.didastone.com`：接口服务
- `admin.didastone.com`：后台
- `test.didastone.com`：测试环境

这里的 `www`、`api`、`admin`、`test`，本质上就是 **主机记录**

![](assets/07%20云服务器/file-20260306202547801.jpg)

那么不同的主机记录，最终对应出来的完整域名就是：
- `www` ：www.didastone.com
- `api`：api.didastone.com
- `mail`  mail.didastone.com
- `@`：`didastone.com` ，也就是**根域名 / 裸域名**
- `*`：最终就是：`*.didastone.com` ，表示**泛解析**，匹配任意未单独配置的子域名，比如 `abc.didastone.com`
