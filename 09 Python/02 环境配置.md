# 02 环境配置

关联：[[01 Python语法]] | [[03 框架开发]] | [[Python语法/模块与包（import）]]

这部分不讲“语法”，讲你作为前端工程师最容易在 Python 上踩坑的地方：

- Python 版本管理（多版本并存、项目锁定版本）
- 虚拟环境（隔离依赖，避免全局污染）
- 依赖管理（安装/锁定/同步/复现）
- 开发体验（VS Code、Jupyter、格式化/检查工具）

> [!tip] 推荐路线（大多数前端转 Python 最省心）
> - 版本管理：`pyenv`（像 nvm 一样管理多个 Python）
> - 依赖与虚拟环境：`uv`（快、现代、适合应用开发）
> - 只有当你需要科学计算/复杂二进制依赖（例如 CUDA、MKL）时，再考虑 `conda`

## 你需要建立的 3 个核心心智模型

### 1) Python 版本 != 虚拟环境

- **Python 版本**：解释器本身（3.10/3.11/3.12...），对应 Node 的版本
- **虚拟环境**：在某个 Python 版本之上，隔离安装第三方包的目录（类似 node_modules + lockfile 的组合体，但实现方式不同）

### 2) “可复现”是工程第一目标

团队协作时你最关心的不是“我本地跑起来了”，而是：

- CI 能跑起来
- 同事 clone 项目也能跑起来
- 三个月后你自己也能复现

这就是为什么我们会强调：`pyproject.toml` + lockfile + 一键 sync（见 [[环境配置/uv]] 与 [[环境配置/pyproject.toml 与依赖管理]]）。

### 3) 不要往全局 Python 装包

除非你非常确定在做什么，否则：

- 不要 `pip install` 到系统 Python
- 不要把项目依赖装到全局

Python 世界里，“全局装包导致项目互相污染”比前端常见得多。
