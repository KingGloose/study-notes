# Vibe Writing 写作工作流

以人为主体的AI辅助写作系统。你决定讲什么、怎么讲，AI提供建议和执行。

## 核心流程

```
学习阶段 → 结构阶段 → 写作阶段 → 成稿阶段
(对话激发)  (全局规划)  (迭代优化)  (串联成稿)
```

## 快速开始

用户说出写作主题后，自动进入学习阶段。支持以下切换命令：
- `/学习` - 进入学习模式，通过对话生成知识卡片
- `/结构` - 进入结构模式，规划文章框架
- `/写作` 或 `/迭代` - 进入写作模式，深化和润色内容
- `/成稿` - 进入成稿模式，串联所有卡片生成完整文章

## 阶段详解

### 阶段1：学习（对话激发）

**目标**：通过苏格拉底式对话，帮用户梳理思路，生成知识卡片。

**流程**：
1. 用户提出主题（如"我想写关于Claude和GPT的区别"）
2. 提问引导，挖掘用户真实想法和独特见解
3. 每4轮对话后，整理生成一张知识卡片
4. 知识卡片包含：核心观点、支撑论据、用户原话金句

**提问技巧**：
- "你为什么对这个话题感兴趣？"
- "你觉得大多数人对此有什么误解？"
- "能举个具体的例子吗？"
- "如果只能告诉读者一件事，你会说什么？"

**知识卡片模板**：见 [references/card-templates.md](references/card-templates.md)

### 阶段2：结构（全局规划）

**目标**：分析知识卡片，提供结构方案，让用户决定文章框架。

**流程**：
1. 汇总所有知识卡片
2. 提供2-3种结构方案（如：问题-方案型、对比型、递进型）
3. 用户选择后，拆分价值点到具体节点
4. 生成输出卡片（每个节点一张）

**结构方案类型**：见 [references/structure-patterns.md](references/structure-patterns.md)

**输出卡片模板**：
```markdown
## [节点标题]

**核心论点**：一句话概括

**展开要点**：
- 要点1
- 要点2

**素材引用**：来自知识卡片#X

**预估字数**：XXX字
```

### 阶段3：写作（迭代优化）

**目标**：逐卡片深化内容，支持迭代和润色。

**流程**：
1. 用户选择要处理的输出卡片
2. 可选操作：
   - 直接提问深入讨论
   - 输入"整理"→ 整合对话到卡片
   - 输入"润色"→ 事实核查 + 语言优化

**润色检查项**：
- 事实准确性（标注需核实的声明）
- 逻辑连贯性
- 语言流畅度
- 去除冗余表达

### 阶段4：成稿（串联输出）

**目标**：将所有输出卡片串联成完整文章。

**流程**：
1. 按结构顺序排列所有输出卡片
2. 添加过渡段落，确保行文连贯
3. 生成开头（hook）和结尾（call-to-action）
4. 输出最终成稿

**成稿增强选项**：
- 添加小标题
- 生成摘要
- 提取金句用于社交媒体

## 文件管理

工作过程中自动创建以下文件结构：

```
00 学习/项目名/
├── 项目信息.md      # 状态追踪
├── 对话记录.md      # 学习阶段对话
├── 知识卡片/        # 学习产出
│   ├── 卡片01.md
│   └── ...
├── 输出卡片/        # 结构产出
│   ├── 01-开篇.md
│   └── ...
└── 最终成稿.md      # 成稿产出
```

## 关键原则

1. **用户主导**：所有决策点都由用户确认，AI只提供选项
2. **渐进式**：每个阶段都有明确产出，可随时暂停
3. **可追溯**：所有内容都能追溯到原始对话
4. **灵活跳转**：可跳过任何阶段，直接进入需要的阶段

## 常见场景

**场景A**：从零开始写文章
→ 完整走完四个阶段

**场景B**：已有想法，需要组织
→ 跳过学习，直接 `/结构`

**场景C**：已有初稿，需要优化
→ 跳过学习和结构，直接 `/迭代`

**场景D**：只想头脑风暴
→ 只用学习阶段，导出知识卡片即可
