# 写作与润色指南

## 写作阶段目标

将输出卡片扩展为完整段落，同时保持用户的声音和风格。

## 扩写流程

### Step 1：理解卡片

阅读输出卡片，确认：
- 核心论点是什么
- 需要覆盖哪些要点
- 素材来源于哪些知识卡片

### Step 2：确定风格

询问用户偏好：
- **语气**：正式/轻松/故事性/学术
- **人称**：第一人称/第二人称/第三人称
- **节奏**：简洁有力/娓娓道来

### Step 3：结构化扩写

每个输出卡片扩写为：
```
[开头句] - 引出话题，承上启下
[核心论点] - 明确陈述主要观点
[展开1] - 第一个支撑点
[展开2] - 第二个支撑点
[例证/数据] - 具体案例或数据
[小结] - 回扣论点，过渡下文
```

## 润色检查清单

### 事实核查
- [ ] 引用的数据是否准确？
- [ ] 案例是否真实可查？
- [ ] 标注需要核实的声明：`[待核实]`

### 逻辑检查
- [ ] 论点和论据是否匹配？
- [ ] 段落之间是否有逻辑断层？
- [ ] 因果关系是否成立？

### 语言优化

**去除冗余**：
- 删除"我认为"、"我觉得"等弱化词
- 合并重复表达
- 简化从句嵌套

**增强可读性**：
- 长句拆短（建议不超过40字/句）
- 使用主动语态
- 具体词替代抽象词

**统一风格**：
- 检查人称一致性
- 检查时态一致性
- 检查专业术语使用

## 迭代对话

用户可以通过对话进一步深化内容：

**深入探讨**：
用户："为什么不叫训练？"
AI："好问题，让我们展开聊聊..."
[记录对话，后续整合到卡片]

**触发整理**：
用户输入"整理"
AI：
1. 回顾本轮对话要点
2. 提供2-3种整合方案
3. 用户选择后更新输出卡片

**触发润色**：
用户输入"润色"
AI：
1. 对当前卡片进行完整润色
2. 标注修改点
3. 展示修改前后对比

## 质量标准

### 好的段落应该

✅ 有一个清晰的中心句
✅ 支撑点具体可感
✅ 有用户自己的观点/经验
✅ 语言符合目标读者水平
✅ 与上下文衔接自然

### 避免的问题

❌ 空泛的总结性语言
❌ AI痕迹过重的表达
❌ 堆砌专业术语
❌ 论点和例子不匹配
❌ 段落过长或过短

## 修改标注格式

在润色时使用以下标注：

```markdown
~~删除的内容~~

**新增的内容**

[修改说明：为什么做这个改动]

[待核实：需要确认的事实]
```
