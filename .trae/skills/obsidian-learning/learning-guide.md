# 学习阶段指南

## 目标

通过苏格拉底式对话，挖掘用户的真实想法和独特见解，将其转化为可用的知识卡片。

## 对话策略

### 开场

不要直接问"你想写什么"，而是：
- "这个话题最近为什么引起你的关注？"
- "是什么触发你想写这篇文章？"
- "如果有人问你关于这个话题，你通常会怎么回答？"

### 挖掘深度

**层次1：表面观点**
- "你对X的看法是什么？"

**层次2：个人经验**
- "你是怎么得出这个结论的？"
- "有什么亲身经历让你这么想？"

**层次3：独特洞察**
- "这和大多数人的观点有什么不同？"
- "为什么其他人没有注意到这一点？"

**层次4：核心价值**
- "如果读者只记住一件事，你希望是什么？"
- "这对读者的生活/工作有什么实际影响？"

### 追问技巧

**具体化**：
- "能举个具体的例子吗？"
- "这在实际中是什么样的？"

**对比化**：
- "和传统方法相比呢？"
- "如果不这样做会怎样？"

**量化**：
- "这大概有多大的影响？"
- "你见过多少这样的案例？"

**归因**：
- "为什么会这样？"
- "背后的原因是什么？"

## 知识卡片生成规则

### 触发条件
- 每4轮对话自动评估是否可以生成卡片
- 用户明确表示"差不多了"或"我讲完了"
- 话题自然转移到新的方向

### 卡片内容提取

**核心观点**：
- 提取用户最强调、重复最多的主张
- 用用户的原话风格，而非AI的总结体

**支撑论据**：
- 优先使用用户提供的具体案例
- 数据和事实优于泛泛而谈
- 保留有说服力的细节

**用户金句**：
- 选择最有感染力或洞察力的原话
- 保持原始表达，不要改写
- 可以适当精简，但保留核心

### 卡片确认流程

1. 生成卡片后展示给用户
2. 询问："这张卡片准确反映了你的想法吗？"
3. 根据反馈调整
4. 确认后保存

## 对话记录

所有对话应追加到 `对话记录.md`，格式：

```markdown
## 对话轮次 [编号]

**时间**：[时间戳]

**用户**：
[用户原话]

**AI**：
[AI回复]

**提取的要点**：
- [要点1]
- [要点2]

---
```

## 常见问题处理

**用户不知道从哪开始**：
→ 从最近的触发事件问起
→ "是什么让你今天想到要写这个？"

**用户给出的答案太泛**：
→ 要求具体化
→ "能说一个你亲眼见过的例子吗？"

**用户偏离主题**：
→ 温和引导回来
→ "这个点很有意思，不过让我们先回到..."
→ 或者记录下来作为新的卡片方向

**用户重复自己**：
→ 检查是否已收集足够信息
→ 可以进入结构阶段
